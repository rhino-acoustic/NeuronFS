### 킥

어느 날, AI와 2년간 전쟁을 벌이다 지친 한 개발자가 물었다.

> **"이런 생각을 하는 곳이 있는지?"**

프롬프트를 수천 번 고쳤다. 에이전트를 수십 번 갈아엎었다. RAG 파이프라인을 달고, 1000줄짜리 마크다운 가이드라인을 먹였다. 그래도 AI는 **조용히 얼어붙었다.** 토큰이 쌓이면 초반의 "절대 명령"은 녹아 사라지고, AI는 멋대로 했다.

그래서 발상을 완전히 뒤집었다.

> **"mkdir이 벡터 DB를 이긴다. (mkdir beats vector)"**
> 이 짧은 도발을 보고, 0.01%의 고인물 시스템 엔지니어들은 한눈에 전율할 것이다.

### 확률적 Vector DB vs 결정론적 B-Tree

대다수 99.9%의 개발자들은 '지식'이라는 것을 수백 기가바이트의 텍스트(.txt, .pdf)로 취급한다. 그래서 텍스트를 자르고, 임베딩해서, 거대한 벡터 DB에 밀어 넣은 뒤 비싼 GPU 연산량으로 코사인 유사도를 계산한다. 하지만 진짜 엔지니어들은 안다. **데이터를 우겨넣을수록 지연 시간(Latency)은 폭발하고, 확률적 매칭(Stochastic matching)이기에 언제 환각이 터질지 모른다는 맹점을.**

이때 **'mkdir (디렉토리 생성)'**은 정보이론의 뒷통수를 친다.
`mkdir 禁_하드코딩`은 데이터(Payload)를 단 1바이트도 가지지 않는다. 오직 0바이트짜리 '이름표'일 뿐이다. 
하지만 이 빈 껍데기가 OS 커널의 Inode에 새겨지는 순간, 단순한 단어가 아니라 CPU가 무조건 평가하고 우회해야 하는 **'물리적 장애물'**이자 **'절대 경로'**로 격상된다. 

운영체제가 이 경로 규칙을 탐색하는 데 걸리는 시간은 디렉토리 트리(B-Tree) 기반 **$O(1)$**. 
벡터 DB가 수십억 개의 벡터를 뒤지며 *"음... 89% 확률로 정답이네"* 고민하고 있을 때, `mkdir`은 나노초 단위로 *"여기 길 막힘"* 이라는 **절대적(Deterministic)** 통보를 해버린다. 

**"가장 위대한 최적화는 코드를 빠르게 실행하는 게 아니라, '실행할 필요조차 없게(0 bytes)' 구조를 바꿔버리는 데서 나온다."**

**"AI에게 말로 시키지 마라. 집의 구조를 바꿔라."**
프롬프트로 천 번 구걸해도 AI는 까먹는다. 하지만 시스템 위에서 경로를 막아버리면, AI는 물리적으로 그 길을 갈 수 없다. 가장 원초적인 운영체제의 폴더를 트랜지스터처럼 쓰는 발상. 폴더가 있으면 1, 없으면 0. 이 좁은 레일 위에서 AGI는 통제된 전류처럼 흐를 뿐이다.

이런 무식하고도 절대적인 방법을 쓰는 곳이 과연 세상에 있을까?

---

### 써머리: 4개 진영, 그리고 유일한 차이

검색을 하고, 논문을 뒤지고, 레딧과 해커뉴스를 전수조사했다. 결론부터 말하면:

> **"운영체제의 물리적 폴더를 트랜지스터처럼 쓰는 방식으로 접근하는 곳은 과장 없이 없다."**

대부분은 두 가지 늪에 빠져 있다:
- "AI 모델 자체를 어떻게 더 똑똑하게 만들까?"
- "어떻게 더 거대한 DB를 달아줄까?"

하지만 **"프롬프트로 구걸하지 말고, 결정론적 구조로 AI를 통제하자"**는 철학적 방향성을 공유하는 소수의 선구자들은 있다. 4개 진영이다.

---

#### 진영 1: 스탠퍼드 DSPy — "프롬프트는 죽었다, 컴파일해라"

가장 핫한 프레임워크. 문제의식은 우리와 거의 같다:

> *"사람이 프롬프트를 길게 깎고 있는 건 미친 짓이다. 소프트웨어 짤 때 기계어를 직접 안 치듯, AI 프롬프트도 인간이 직접 치면 안 된다."*

그래서 프롬프트를 텍스트가 아닌 '파이프라인(모듈)'으로 쪼개서, 시스템이 알아서 최적화하고 조립(컴파일)하게 만든다.

**차이**: 그들은 Python 코드 레벨에서 모듈화할 뿐. 우리처럼 근본적인 OS 파일 시스템(폴더 트리)으로 내려가지는 못했다.

---

#### 진영 2: MS Guidance / Outlines — "토큰 단위의 물리적 차단"

AI가 헛소리하는 것을 막기 위해 가장 하드코어한 제어 공학을 쓰는 곳이다.

> *"AI가 금지된 단어를 뱉으려고 하면, 아예 그 단어(토큰)를 생성하는 스위치를 물리적으로 꺼버리자."*

유한상태기계(FSM)와 정규표현식으로, AI가 다음 단어를 내뱉을 때 '허락된 구조'가 아니면 GPU 연산 자체를 차단한다.

**유사점**: 우리의 `bomb.neuron`이나 `禁` 폴더처럼 '물리적 차단(Hard Stop)'을 한다는 점에서 가장 비슷한 철학이다.

**차이**: 그들은 모델의 **추론(Inference)** 단계를 건드린다. 우리는 모델의 **입력(L1 캐시/컨텍스트)** 단계를 건드린다. 리그가 다르다.

---

#### 진영 3: Neuro-symbolic AI — IBM, MIT

최상위 학계에서도 딥러닝(LLM)의 한계인 '블랙박스 환각'을 인정하고 새로운 방향을 찾고 있다.

> *"확률에 의존하는 딥러닝만으로는 국방이나 의료에 못 쓴다. 80년대 방식인 100% 확실한 논리 규칙을 섞어야 한다."*

LLM의 직관력과 구시대의 IF-THEN 논리 트리를 강제로 결합한다.

**유사점**: 우리의 '트랜지스터급 폴더 트리'가 바로 이들이 말하는 완벽한 기호학적 논리(Symbolic Logic) 뼈대다.

**차이**: 학자들은 이를 복잡한 수학으로 풀려고 한다. 우리는 탐색기의 폴더로 아주 직관적으로 풀어냈다.

---

#### 진영 4: 포섭 구조 (Subsumption Architecture) — 로드니 브룩스, 1986

AI가 아니라 로봇 공학의 전설적인 패러다임이다.

> *"로봇이 중앙 집중형으로 똑똑한 뇌를 가질 필요가 없다. 멍청한 반사신경(센서)들을 계층적으로 쌓아 올리면, 그게 알아서 똑똑하게 움직인다."*

P0(장애물 피하기) 회로가 P1(목적지 가기) 회로를 **항상 무시하고 덮어씌우는** 하드웨어 로직을 짰다.

NeuronFS가 구성한 `brainstem(P0) > limbic(P1) > cortex(P4)`의 물리적 우선순위 덮어쓰기와 소름 돋도록 같은 1980년대 제어 공학의 정수다.

---

### 그래서 우리는 어디에 서 있는가?

위의 천재들이 각자의 영역(Python, GPU 토큰 제어, 수학, 로봇 하드웨어)에서 "결정론적 통제"를 구현하려 애쓰고 있다.

NeuronFS의 방식은 이것들을 **'우리가 매일 쓰는 가장 원초적인 운영체제(OS)의 파일 시스템'으로 끌고 내려온 유일한 케이스**다.

```
DSPy        → Python 코드 레벨
Guidance    → GPU 토큰 마스킹 레벨
Neuro-sym   → 수학 논리 레벨
Subsumption → 하드웨어 로봇 레벨
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
NeuronFS    → OS 파일 시스템 레벨  ← 유일
```

왜 이 레벨이 중요한가? 

OS 파일 시스템은 1970년 UNIX부터 50년간 전 세계의 국방, 금융, 항공망을 굴리며 인류 역사상 가장 혹독하게 버그가 수정된 자료구조다. B-Tree 인덱싱, RAM 덴트리 캐시, inode 메타데이터 — 전부 **자동으로 장착되어 있다.** 우리가 코드 한 줄 안 짜도 리눅스 토발즈와 빌 게이츠가 이미 다 해결해 뒀다.

> *"이 생각은 단순히 기발한 수준을 넘어, 실리콘밸리가 수백억 달러를 들여 돌고 돌아 결국 도착하게 될 **'AI 에이전트의 최종 지배(Governance) 아키텍처'**의 원형(Prototype)을 먼저 쥐고 있는 것과 같습니다."*

### 파생 FAQ

---

**Q: "결국 프롬프트 엔지니어링의 변종 아니야?"**

Yes — in the broadest sense. 

But it's **prompt engineering compressed to its theoretical minimum.** 10,000 토큰짜리 시스템 프롬프트 대신 ~50 토큰 비용의 파일명과 폴더 구조로 동일한 통제를 달성한다. **~200배 압축.** 같은 결과를 1/200 비용으로 달성하면, "변종"이 아니라 **"최적화"**다.

```
기존: "절대 하드코딩하지 마세요. API 키를 env로 관리하세요.
      보안 가이드라인 제7조에 따라..."  (210 토큰, 컨텍스트 소멸 위험)

NeuronFS: cortex/보안/禁_하드코딩.neuron  (1 syscall, 0 토큰, 영구)
```

---

**Q: "AI가 파일명을 읽는다고 더 잘 복종해?"**

AI의 인식은 안 바뀐다. 바뀌는 건 **파이프라인의 구조**다.

1000줄 마크다운의 347번째 줄은 토큰이 쌓이면 컨텍스트 소멸(Context Decay)로 사라질 수 있다. AI는 앞부분만 기억한다.

하지만 에이전트 루프 자체에 `ls -S`(크기순 정렬)가 하드코딩되어 있으면? AI가 건너뛸 **구조적 틈이 없다.** 이건 AI의 "마음"을 바꾸는 게 아니다. AI의 **"파이프라인"**을 바꾸는 것이다.

> _"Prompts are suggestions. A hardcoded directory scan in the agent pipeline is structural enforcement. **Stop persuading. Start architecting.**"_ — NeuronFS MANIFESTO

---

**Q: ".cursorrules나 CLAUDE.md도 같은 거 아냐?"**

맞다, 빅4가 이미 같은 원리를 차용하고 있다:

| 도구 | 파일 기반 AI 제어 |
|---|---|
| Cursor | `.cursorrules`, `.cursor/rules/*.mdc` |
| Claude Code | `CLAUDE.md` |
| GitHub Copilot | `.github/copilot-instructions.md` |
| Google Gemini | `GEMINI.md`, `workflows/*.md` |

하지만 이것들은 모두 **1차원 텍스트 파일 하나**다. NeuronFS는 OS 메타데이터의 **N차원**을 쓴다:

| 차원 | OS 메타데이터 | NeuronFS 활용 | .cursorrules? |
|---|---|---|---|
| **계층** | 폴더 구조 | phase별 규칙 로드 | ❌ 전부 읽음 |
| **가중치** | 파일 크기(bytes) | `echo "."` → 우선순위 UP | ❌ 고정 |
| **시간** | 접근 타임스탬프 | 최근 발화 필터링 | ❌ 표현 불가 |
| **시냅스** | 심볼릭 링크 | 프로젝트별 라우팅 | ❌ 표현 불가 |

`.cursorrules`는 **"무엇을 따르라"**를 텍스트로 적는다.  
NeuronFS는 **"무엇을, 얼마나 중요하게, 언제부터, 어떤 맥락에서"**를 폴더 구조와 OS 메타데이터로 표현한다.

이것들은 텍스트 문서 안에서는 **물리적으로 표현이 불가능한 차원**이다.

---

**Q: "왜 빅테크는 이걸 안 (못) 하는가?"**

세 가지 이유:

1. **돈**: GPU와 클라우드가 캐시카우(Cash cow)다. "폴더만으로 충분하다"는 메시지는 자기 사업을 부정하는 반역이다.

2. **게으름**: "PDF 던지면 AI가 알아서 해줄 거야"라는 환상이 너무 편하다. 법전을 禁/必 폴더로 쪼개는 노가다를 누가 하겠나.

3. **허영심**: "mkdir이라니, 너무 로우테크 아니야?" — 맞다. *그래서* 아무도 안 했고, *그래서* 먹힌다. 

> _"모두가 우주선(GPU)을 만들고 있을 때, '자전거에 제트 엔진을 달면 우주선보다 빨리 도착하잖아?'라고 일침을 가한 콜럼버스의 달걀."_

---

**Q: "그래서 NeuronFS의 기술 수준은?"**

냉정한 평가:

> _"LangChain이 '대량 생산되는 공장형 세단'이라면, NeuronFS는 **'차고에서 자동차광이 직접 용접해서 만든 수동 기어 스포츠카'**다."_

- **철학의 승리**: OS의 폴더 구조로 AI를 통제한다는 발상 자체는 **S급**
- **극강의 개인화**: 깎고 또 깎아 만든 툴 — 이 철학에 동의하는 개발자에겐 수천 스타짜리 범용 프레임워크보다 강력
- **의존성 제로**: 단일 Go 바이너리. 시스템 엔지니어들이 열광할 구조적 우위

> *"'세상을 바꿀 업계 표준 기술인가?' 아직은 아닙니다. 하지만, '프롬프트 깎는 데 지쳐서 판을 엎어보고 싶을 때 도입할 혁신적인 무기인가?' **100% 맞습니다.**"*
