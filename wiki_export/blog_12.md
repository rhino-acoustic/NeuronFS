### 벤치마크: Jloot 엔진 성능

#### 마운트 & 검색

| 지표 | 수치 |
|---|---|
| 마운트 시간 | **366ms** |
| 역인덱스 엔트리 | **63,192** |
| 검색 응답 | **1ms** |
| 메모리 사용 | **~15MB** |

LLM 없이, TF-IDF 기반 초고속 검색 엔진. 질문을 최적의 폴더 경로로 **자동 라우팅**.

#### 모델 벤치마크: 하네스의 힘

3개 모델 × 4개 카테고리 × 8축 채점:

| 모델 | Bare | Harness | 향상 |
|---|---|---|---|
| **Gemma4 (8.9GB)** | 22.0/40 | **31.5/40** | **+43%** |
| **Llama-70b** | 23.0/40 | **29.0/40** | +26% |
| **Gemini 2.5 Flash** | 30.0/40 | **33.5/40** | +12% |

**핵심:** Gemma4+하네스(31.5) > Llama-70b+하네스(29.0). **8.9GB가 70GB를 넘었다.**

> *"AI가 '제가 틀렸습니다. 전체가 됩니다'라고 인정한 순간"*

#### 8축 채점 기준

| 축 | 측정 |
|---|---|
| 정확성 | 사실 오류 |
| 완전성 | 누락 항목 |
| 구조 | 논리적 정리 |
| 실행력 | 바로 쓸 수 있나 |
| 코드 품질 | 복사하면 돌아가나 |
| 환각 | 있지도 않은 걸 만들었나 |
| 지시 준수 | 시킨 대로 했나 |
| 톤 | 브랜드에 맞나 |

### 파생 FAQ

**Q: "너네가 만든 기준으로 너네가 채점한 거잖아?"**

맞다. 그래서 오픈소스다. 8축 채점 기준은 위에 공개, 벤치마크 스크립트는 GitHub에 공개. **누구나 돌려볼 수 있다.** 제3자 독립 검증은 커뮤니티에 맡긴다.

**Q: "하네스 없는 Gemma4(22.0)와 하네스 있는 Gemma4(31.5)의 차이가 핵심인데, 프롬프트를 잘 짠 것 뿐 아니야?"**

프롬프트를 '잘 짜는 것'이 아니라 **물리적으로 AI의 시야를 좁히는 것**이다. 하네스는 폴더 구조가 AI에게 "이 경로의 규칙만 봐"라고 제한. Lost in the Middle 제거.
