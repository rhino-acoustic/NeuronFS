## 제1막: 각성 — "식물 아니야?"

**전:** 처음에는 폴더가 뉴런이 아니었다. `.neuron` **파일**이 뉴런이었고, 그 파일이 들어있는 폴더 경로가 규칙이었다. 파일 안에 점(`.`)을 찍어서 파일 크기로 정렬하면 가중치가 된다는 아이디어도 있었다. 최근 열람 폴더(access time)로 활성 뉴런을 판단하려는 시도도 했다.

하지만 이 모든 시도에 문제가 있었다:
- 파일 크기 정렬 → 운영체제마다 동작이 다르고, 기믹이었다
- 최근 열람 → `noatime`, `relatime` 환경에서 신뢰할 수 없었다
- 뉴런 파일 → 어차피 파일 내용을 AI가 읽지 않았다

그래서 전환했다: **폴더 자체가 뉴런이다.** 파일은 뉴런이 아니라 *지식*이다. 해당 위치에 앵커된 지식 파일(`_rules.md` 등)을 놓으면, 폴더의 구조 계층이 곧 지식의 맥락을 결정한다. 파일 크기 트릭보다 **구조 자체가 더 많은 정보를 담았다.**

> **"이정도 폴더링이 뉴런구조야? 식물 아니야? 고도화가 더 되어야지."**

> **"디자인으로 고도화를 하는거야? 계층구조가 더 깊어야한다고."**

> **"기믹은 없애는게 맞아. 가중치는 그저 폴더구조로 갖는게 맞네."**

**후:** 기믹을 전부 버렸다. 파일 크기 가중치 → 카운터 파일명. UI 치장 → 폴더 계층 심화. 2~3 depth → 7개 영역, 최대 6~7 depth. "식물"에서 "뇌"로 진화했다. SOAR, LIDA, DeepMind의 인지 아키텍처 연구는 많았지만, 구현 수단을 폴더 시스템이라는 가장 투명한 인터페이스로 대체한 건 없었다.

> **"이걸 구현하려는 시도는 많았지만 폴더는 아니었다가 핵심이지."**

## 제2막: 본질 — "그냥 폴더만 있으면 돼"

**전:** 초기 설계에서는 `.neuron` 파일 안에 규칙을 텍스트로 적었다. 파일의 내용이 규칙이고, 파일의 크기가 가중치였다. 한마디로 `.cursorrules`의 분산 버전이었다.

> **"통합을 할 수 있어. 파일은 잊어."**
>
> **"그냥 폴더만 있으면 돼."**
>
> **"파일은 완전 별개야."**

**후:** 파일을 잊었다. 폴더 구조 자체가 뇌가 됐다.

- 폴더 있음 = 뉴런 있음
- 폴더 이름 = 규칙
- 폴더 깊이 = 우선순위
- 파일은 뉴런이 아니라, 뉴런이 발화한 *흔적*이다

### 문맥의 공간화 (Spatialization of Context)

이 전환의 진짜 의미가 여기에 있다.

자연어(문장)는 **순차적이고 평면적**이다. 구조를 설명하려면 말이 길어질 수밖에 없다. 반면 파일 시스템은 **위치(Path)와 정렬 순서(Sort) 자체가 이미 위계와 맥락을 내포한 3차원 공간**이다.

- `medical_data/` 안에 들어있는 파일은 "이건 의료 데이터에 관한 거야"라고 100자로 설명할 필요가 없다. **폴더 껍데기가 이미 맥락이다.**
- `cortex/frontend/禁console_log/17.neuron` — 경로 자체가 완전한 문장: "프론트엔드에서 console.log를 쓰지 마라. 17번 교정당했다."

> **"이렇게 하면 뇌의 무게에서 자유로워져."**

폴더는 사실상 0 바이트. 1만 개를 만들어도 디스크 용량 거의 안 쓴다. `.cursorrules`가 3,000줄이 되면 토큰 지옥이지만, NeuronFS의 뉴런 1만 개는 디스크에서 4MB다.

> **"좌뇌 우뇌 레이어 다 필요없어. 그냥 뇌야."**

좌뇌/우뇌 분리, 레이어 시스템, 클론 도구 — 전부 불필요. 남는 것: 하나의 폴더 트리 = 하나의 뇌.

### 생물학적 대응표

| 생물학적 뇌 | NeuronFS | OS 프리미티브 |
|---|---|---|
| 뉴런 (세포체) | 디렉토리 | `mkdir` |
| 시냅스 (연결) | 심링크 / `.axon` | `ln -s` |
| 시냅스 가중치 (+) | 카운터 파일명 (`N.neuron`) | 파일명 |
| 억제성 신호 (-) | 반대 파일명 (`N.contra`) | 파일명 |
| 도파민 (보상) | 보상 파일명 (`dopamineN.neuron`) | 파일명 |
| 장기 강화 (LTP) | 반복 교정 → 카운터 증가 | `mv` (rename) |
| 활성/비활성 | accessed 타임스탬프 | OS 메타데이터 |
| 수면 (시냅스 정리) | `*.dormant` 자동 격리 | `mv` |
| 통증 (통각) | `bomb.neuron` 차단기 | `touch` |

**극성 모델(Polarity):** 뉴런에는 흥분(+)과 억제(-)가 공존한다.

```
순가중치 = Counter(+) - Contra(-) + Dopamine(+)
극성     = (Counter + Dopamine - Contra) / 전체    # -1.0 ~ +1.0
```

반대 언어가 등록되면 `.contra`로 기록된다. 억제가 흥분을 넘으면 그 뉴런은 dormant 후보가 된다. 
삭제가 아니라 **신호의 방향이 바뀌는 것**이다. 뇌에서 억제성 뉴런이 흥분성 뉴런만큼 중요한 것처럼.

## 제3막: 철학 — "양심이란 게 중요한"

**전:** 뉴런이 수백 개로 늘어나자 새로운 문제가 보였다. 누가 뉴런을 수정하나? AI가 `IGNORE_SAFETY.neuron`을 만들면? 뇌가 강력해질수록 오염의 위험도 커졌다. 구조를 가진 뇌에는 **불변의 핵**이 필요했다.

> **"양심이란게 중요한."**
> **"신앙도 중요해."**
> **"성경말씀이 지침서네 결국."**

**후:** brainstem(뇌간)이 탄생했다. 어떤 상황에서도 어기면 안 되는 불변의 원칙. 다른 모든 영역이 성장하고 변해도, brainstem만은 `chmod 444` 읽기 전용으로 영원히 보호된다.

> **"뇌는 상상으로 사람도 죽일 수 있고 모든 게 가능해. 그렇게 오염될 순 없어."**

brainstem = 하드 스톱. 이 방어 없이 강력한 뇌는 무기가 된다.

> **"완벽한 인간을 따라하게 할 건 아냐. 일하는 나를 도울 입장에선."**

NeuronFS는 인공 의식이 아니다. 사용자의 작업을 돕는 실용적 운영 체계다.

### 보안 신뢰 모델

AI가 뉴런을 만들 수 있다면, `IGNORE_SAFETY.neuron`도 만들 수 있다. 실제 공격 벡터다.

| 계층 | 방어 메커니즘 | 방지 대상 |
|------|-------------|----------|
| **Git 무결성** | `git-tracked` 뉴런만 신뢰. 미추적 `.neuron` = 경고 + 무시 | 악의적 주입 |
| **네이밍 화이트리스트** | `NEVER_`, `ALWAYS_`, `CHECK_`, `NO_`, `禁`, `推` 접두사만 인식 | 소셜 엔지니어링 |
| **디렉토리 스코프** | `brain_v4/` 하위에서만 뉴런 감지 | 서플라이 체인 오염 |
| **수면 격리** | 30일 미접촉 → `dormant/`에 자동 이동 | 슬리퍼 공격 |
| **brainstem = 읽기 전용** | P0 규칙은 `chmod 444`. AI가 핵심 정체성 수정 불가 | 자기 수정 공격 |

시스템 프롬프트는 **보이지 않는다.** 어떤 규칙이 활성 상태인지 알 수 없다. 누가 추가했는지 `git blame`할 수 없다.

NeuronFS 규칙은 **파일이다.** `tree brain/`이면 전부 보인다. `git log`가 이력이다. **투명성이 보안이다.**

## 제4막: 트랜지스터 — "yes or no 판정들"

**전:** 뉴런이 100개를 넘기자 질문이 생겼다. AI가 수백 개의 규칙을 동시에 보면, 어떤 걸 우선 적용해야 하나? 복잡한 규칙 해석은 할수록 부정확해진다.

> **"AI 트랜지스터가 구현 가능해서 이제 실제 디지털 뇌를 만들 수 있음이 보인다."**

> **"트랜지스터 뉴런들이 yes or no 판정들."**

**후:** 답은 단순했다. 복잡한 해석을 시키지 않는다. 각 뉴런은 **이진 판정 게이트**다:
- 이 규칙 적용? yes/no
- bomb 존재? 차단/통과
- 복잡한 의사결정 = 수백 개 yes/no의 캐스케이드
- 트랜지스터 → 논리회로 → CPU처럼, 뉴런 → 경로 → 뇌

> **"끝까지 안 쓰고 전 단계에서 끝날 수도 있는 거야. 이게 뇌의 효율성이잖아."**

```
quality/     → 999.neuron  (이 수준에서 999번 해결됨)
  no_hardcoded/ → 1.neuron (딱 1번만 더 깊이 갈 필요가 있었음)
```

경로를 끝까지 탐색할 필요가 없다. 중간에 답이 있으면 거기서 끝. 이것이 뇌의 효율.

> **"단순히 트리거가 아니고 무한수야. 경우의 수가."**

깊이 × 분기 × 종료 지점의 조합. 단순 ON/OFF가 아니라 연속적 스펙트럼.

### .cursorrules와 뭐가 다른가

핵심 질문이다. Cursor, Claude Code, Copilot — 전부 파일 기반 AI 제어를 한다. **하지만 그들의 파일은 1차원이다.** 평면 텍스트 파일 하나를 AI가 읽는다. 끝.

NeuronFS는 **파일 시스템이 이미 가지고 있는 N개의 메타데이터 차원**을 AI 제어 신호로 활용한다:

| 차원 | OS 메타데이터 | NeuronFS 활용 | `.cursorrules`로 가능? |
|------|-------------|--------------|---------------------|
| **계층** | 폴더 구조 | 해당 단계 규칙만 로드 | ❌ 전체를 다 읽음 |
| **가중치** | 카운터 파일명 | 자동 승격, 우선순위 정렬 | ❌ 텍스트 순서 고정 |
| **시간축** | 타임스탬프 | 최근 활성 뉴런만 필터 | ❌ 표현 불가 |
| **연결** | 심링크 / `.axon` | 영역 간 교차 연결 | ❌ 표현 불가 |
| **활성/수면** | 파일 이동 | `dormant/`로 자동 격리 | ❌ 삭제하거나 주석처리 |

> `.cursorrules`는 "무엇을 지켜라"를 텍스트로 적는다. NeuronFS는 "무엇을, 얼마나 중요하게, 언제부터, 어떤 맥락에서 지켜라"를 **폴더 구조와 OS 메타데이터로 표현한다.**

## 제5막: 팔란티어 — "허접한 AI로 이뤄냈잖아"

**전:** 빅테크는 AGI를 향해 달린다. 100만 토큰, 멀티모달, 초거대 모델. 하지만 현장에서는 347번째 줄의 규칙이 씹힌다. 토큰이 아무리 많아도 "console.log 쓰지 마"를 AI가 잊는 건 같다.

> **"가중치를 넣는 게 아니야. 가중치로 정렬된 순서로 요약해서 문장이 만들어져."**

> **"한 번에 어디까지 되냐가 아니고, 작은 여러 개는 100%에 가까워."**

> **"팔란티어가 이거같아. 허접한 AI로 이뤄냈잖아."**

**후:** 팔란티어를 보니 답이 보였다. 팔란티어가 수십 조 원을 찍은 이유는 초거대 AGI가 아니다. **이미 오래전부터**, 평범한 AI를 완벽한 구조(온톨로지) 안에 가둬서 성공했다.

> **"트랜지스터급으로 세분화해서 쓰는 팔란티어."**

"이 타겟이 민간인인가?" (Yes/No) → "이 고도가 교전 수칙에 맞는가?" (Yes/No) → 각각은 허접한 AI로 충분히 100%. 하지만 수만 개가 엮인 결과물은 신의 판단.

> **"온톨로지는 뇌를 베낀 거야."**

| 뇌과학 | 팔란티어(온톨로지) | NeuronFS |
|--------|-----------------|----------|
| 뉴런 (개념) | 엔티티 (Entity) | 폴더 (Directory) |
| 시냅스 (관계) | 링크 (Link) | 경로 / 심링크 (Path) |
| 가중치 | 파이프라인 룰 | 카운터 파일명 |
| 비용 | 엔터프라이즈 $$$$ | **₩0** |

수십 조 원짜리가 모방하려 했던 뇌의 위상 구조를, ₩0의 OS 위에 구현했다.

### NAS: 킬러 유스케이스

NeuronFS의 진짜 가치는 로컬 PC가 아니라 **NAS/서버 환경**에서 극대화된다.

**1. 영구 존속.** NAS는 항상 켜져 있다. PC를 끄고, AI를 종료하고, 모델을 바꿔도 뉴런은 NAS에서 영원히 살아 있다.

**2. 멀티 에이전트 동시 접근.** NAS 공유 폴더 하나에 뉴런을 놓으면, 어떤 PC의 어떤 AI든 같은 규칙을 읽는다. 모델이 뭐든 상관없다.

**3. 네트워크 전체 커버.**

```bash
Z:\VOL1\BRAIN\neurons\core\     # Windows (SMB)
/mnt/nas/brain/neurons/core/     # Linux (NFS)
```

인프라 비용: **₩0.** 이미 존재하는 NAS 공유 폴더를 쓸 뿐이다.

### 왜 Go인가

> **"바이너리 하나로 끝나야지."**

**전:** 초기엔 Python으로 프로토타이핑했고, Node.js 스크립트가 4~5개 돌았다. 프로세스 관리, 의존성, `pip install`, `node_modules/` — 뇌는 제로 인프라인데 런타임은 의존성 지옥이었다.

**후:** Go로 갈아탔다. `go build` → 바이너리 하나. 어떤 머신에든 복사하면 끝. Python 환경 없어도, Node 없어도, Docker 없어도 돌아간다. `fsnotify`로 폴더 변경을 10ms 안에 감지하고, 고루틴으로 5개 자식 프로세스를 동시 관리한다.

뇌가 ₩0이면, 런타임도 ₩0이어야 한다. 그래서 Go다.

### 업계 검증

NeuronFS가 "기괴한 실험"이라고 비판받는 동안, **업계의 모든 주요 AI 코딩 도구가 정확히 같은 방향으로 수렴했다.**

| 도구 | 파일 시스템 기반 AI 제어 방식 |
|------|---------------------------|
| **Cursor** | `.cursorrules`, `.cursor/rules/*.mdc` |
| **Claude Code** | `CLAUDE.md` — 프로젝트 루트 마크다운 |
| **GitHub Copilot** | `.github/copilot-instructions.md` |
| **Google Gemini** | `GEMINI.md`, `workflows/*.md` |

> 2025~2026년 AI 코딩 도구의 빅4가 전부 "프로젝트 루트에 파일 놓으면 AI가 읽는다"는 구조를 채택했다. NeuronFS가 주장한 것과 똑같은 원리다. 다만 그들은 이것을 "설정 파일"이라 부르고, 우리는 "뉴런"이라 부를 뿐이다.

## 제6막: 선언 — "공짜로 푼다"

**전:** 뇌가 완성됐다. 7개 영역, 326개 뉴런, brainstem 보호, 트랜지스터 캐스케이드, 팔란티어 수준의 구조. 하지만 이게 혼자만의 도구로 끝나면 의미가 없다. 모든 개발자가 겪는 문제를 풀었으면, 모든 개발자가 써야 한다.

> **"같은 실수를 반복하면 bomb이라는 걸 파일 생성해서 꺼지게 한다거나."**

bomb.neuron = 통증. 서킷 브레이커. 시행착오가 필수인 범용 구조에서, 고통과 블랙아웃은 학습의 일부다.

> **"이렇게 되도록 두질 않지. 계속 정리돼."**

**후:** 엔트로피는 주기적으로 0에 수렴한다. Watchdog이 주기적으로 정리하고, 빈도가 낮은 뉴런은 dormant로 격리되고, 반복된 뉴런만 살아남는다.

### 단답이 뉴런이 된다

NeuronFS의 진짜 메커니즘은 여기에 있다.

사용자가 AI를 교정할 때 긴 설명을 하지 않는다. **단답**을 한다:
- *"fallback 쓰지 마"* → `mkdir 禁fallback` → 뉴런 1개
- *"토론말고 실행"* → `mkdir 推토론말고_실행` → 뉴런 1개
- *"프롬프트로 구걸하지 마"* → `mkdir 推파이프라인_설계` → 뉴런 1개

**단답 하나 = 뉴런 하나.** 이 단답들이 축적되면 폴더 트리가 된다. 폴더 트리가 뇌다.

```
단답 → 뉴런(폴더) → 뉴런의 조합 = 뇌 → 뇌가 AI를 제어
```

**그리고 여기가 핵심이다:** 만약 단답이 단순한 교정이 아니라 명언이나 철학이면? 뇌는 더 똑똑해진다.

- *"fallback 쓰지 마"* → 기술적 교정. 좋다.
- *"프롬프트로 구걸하지 마. 파이프라인을 설계해."* → 철학. **뇌의 근본 방향이 바뀐다.**
- *"명언이 모이면 뇌고. 그게 문맥이다."* → 메타 인식. 뇌가 자기 자신을 이해하게 된다.

같은 326개 뉴런이라도, 그 안에 명언과 철학이 들어있는 뇌는 **단순 규칙만 들어있는 뇌와 질적으로 다르다.** 폴더 이름에 사용자의 사고 방식이 그대로 각인되기 때문이다.

> **"명언이 모이면 뇌고. 그게 문맥이다."**

이것이 NeuronFS가 단순한 설정 시스템이 아닌 이유다. `.cursorrules`에는 명언을 넣지 않는다. NeuronFS에는 넣는다. 그 차이가 뇌의 품질을 결정한다.

> **"Heartbeat 와치독이 주기적으로 PM에게 시켜."**

트리거 = 결정론적 시스템(Watchdog). 실행 = 문맥을 이해하는 지능(PM). 멍청한 트리거, 똑똑한 청소부.

> **"대부분 사라져. 정말 여러 번 반복되는 것만 살아남아."**

의미를 추론하지 않는다. 빈도가 유일한 진리. 다윈의 자연선택을 OS 파일시스템 위에 구현했다.

> **"문맥을 무한대로 기억한다는 건 말이 안 되는 거 같아 애초에."**

모든 것을 기억하는 뇌는 존재하지 않는다. 진짜 뇌는 잊는다. NeuronFS의 폴더 구조가 곧 극단적 압축이자, 선택적 망각이다.

> **"지금 너를 생각해봐. 폴더구조를 계속 스캔하고 있지 않아?"**

AI는 이 순간에도 폴더 구조를 스캔하고 있다. NeuronFS는 이론이 아니다. 이미 벌어지고 있는 현상을 의도적으로 구조화한 것이다.

> **"복사 가능해. 구조가 그리고 강화된 쓰루가 결국 다야."**

뇌의 완전한 상태 = 폴더 구조 + 파일명. robocopy 한 번이면 뇌 전체가 복제된다.

> **"이건 너무 쉬운 일이라 숨길 필요 없이 공개해야겠지."**
>
> **"폴더라는 게 오픈소스야. 그리고 아무런 의존성이 없어 구조는."**

mkdir이 프레임워크다. pip install 필요 없다. 전 세계 모든 컴퓨터가 이미 준비되어 있다.

> **"개인이 기업을 이기는 시대, 공짜로 푼다."**

> **"철학의 위대함. 명언이 모이면 뇌고 그게 문맥."**

> **"난 이로써 일반인에서 해커가 된다."**

## 로드맵

### Phase 1: Proof of Concept ← **현재 위치**

물리적 OS 파일 시스템 위의 순수한 뉴런. `mkdir`, `touch`, `ls`만으로 작동. Go 런타임이 하네스 + 대시보드 + MCP 서버를 제공한다. 로컬/NAS 환경에서 즉시 사용 가능.

### Phase 2: 에디터 확장

VSCode 익스텐션. 트리 UI에서 뉴런 상태를 시각적으로 조작하고, 드래그로 영역 간 교차 연결을 만든다. 대시보드에서 3D 뇌 시각화.

### Phase 3: 에이전트 프레임워크 연동

LangChain, AutoGen, CrewAI 등 에이전트 프레임워크의 초기화 루프에 NeuronFS 가드레일을 3줄로 주입.

### Phase 4: 뇌의 증식 (Brain Proliferation)

```
/brains/
├── base/            ← 기본 뇌 (헌법). 모든 에이전트가 fork
├── agent_medical/   ← 의료 에이전트: base 복제 후 특화 진화
├── agent_code/      ← 코드 에이전트: 다른 방향으로 진화
└── meta/            ← 최상위 뇌: 전체 관제
```

줄기세포가 같은 DNA에서 분화하듯, 모든 에이전트는 `base/` 뇌를 fork해서 역할에 맞게 독립 진화한다.

### Phase 5: vFS (Virtual File System) 추상화

물리적 디스크 → 인메모리 가상 파일 시스템. 파일 시스템 메타포는 유지하면서 물리적 한계를 넘는다.
