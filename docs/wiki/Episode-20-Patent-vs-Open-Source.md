### 특허는 함정이다

| | 특허 | 오픈소스 |
|---|---|---|
| 방어 범위 | 좁음 (폴더 구조 특허? 선행기술 풍부) | 생태계 전체 |
| 비용 | 수천만 원 (출원+유지) | 0원 |
| 소요 시간 | 2-3년 | 즉시 |
| 대기업 대응 | 대충 변형해서 우회 | **우리 위에서 빌드해야 함** |
| 시장 영향 | 없음 | **카테고리 장악** |

```
특허: "이건 내 기술이다" -> 법적 싸움 -> 스타트업 레벨에서 무의미
공개: "이건 새 카테고리다" -> 커뮤니티 선점 -> 표준 규격이 됨
```

### 역사가 증명하는 패턴

- **리눅스**는 특허를 안 냈다. **리눅스가 됐다.**
- **워드프레스**는 특허를 안 냈다. **워드프레스가 됐다.**
- **Docker**는 특허를 안 냈다. **Docker가 됐다.**
- **Kubernetes**는 특허를 안 냈다. **Kubernetes가 됐다.**

**공통점:** 기술이 아니라 카테고리를 만든 자가 이겼다.

> *"기존에 없는 기술은 특허를 이길 수 없다. 대신 카테고리를 만들어 종교의 창시자가 돼라."*

### 방어적 공개 (Defensive Publication)

NeuronFS는 **제3의 길**을 선택한다:

```
[특허]  -> 비용 높음, 범위 좁음, 대기업에 무의미
[비공개] -> 선점 불가, 누가 먼저 만들었는지 증명 불가

[방어적 공개] -> GitHub 커밋 히스토리 = 불가역적 Prior Art
  -> 누군가 나중에 특허를 내려 해도, "이미 공개된 기술"이라 등록 불가
  -> 비용: 0원
  -> 효과: 영구적
```

GitHub의 커밋 타임스탬프가 법적 우선권(Prior Art)을 증명한다. NeuronFS의 모든 위키 에피소드가 **방어적 공개 문서**다.

### AGPL-3.0: 오픈코어의 방패

```
[무료] 엔진 (Go 바이너리, AGPL-3.0)
  -> AGPL 조건: 수정 코드 반드시 공개
  -> 대기업이 포크해서 클로즈드로 쓸 수 없음
  -> 상용 사용 시 별도 라이선스 구매 필요 (연매출 1M USD 이상)

[유료] 뇌 (.jloot 카트리지)
  -> AGPL 대상 아님 (별도 상용 라이선스)
  -> 암호화된 전문가 지식
  -> DRM 구독
  -> Brain Marketplace 수수료
```

> *"코드는 AGPL로 열어서 커뮤니티를 키우고, 브레인 카트리지는 DRM으로 잠가서 수익화한다. 엔진은 리눅스처럼 퍼지고, 뇌는 iOS 앱처럼 팔린다."*
