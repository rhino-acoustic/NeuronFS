NeuronFS
시작 가이드

코딩을 몰라도 됩니다. AI에게 규칙을 텍스트로 부탁하면 60%만 따릅니다.
폴더로 만들면 100% 따릅니다. 5분이면 끝납니다.

🤔 NeuronFS가 뭔데?

학교에서 선생님(AI)한테 "숙제 안 하면 안 돼!"라고 말하면 가끔 까먹죠?
하지만 교실 문에 "숙제 안 하면 출입 금지"라고 붙여놓으면? 절대 못 지나갑니다.

NeuronFS는 AI의 교실 문에 규칙을 붙여놓는 도구입니다.

🚫

= 절대 하면 안 돼!
✅
必 (필수)
= 반드시 해야 해!

준비물 확인
시작하기 전에 이것만 있으면 됩니다

NeuronFS를 쓰려면 딱 2가지만 필요합니다:

  • ✓
    Git — 코드를 다운로드할 도구
    터미널에 git --version 쳐서 숫자가 나오면 OK
  • ✓
    Go (1.22 이상) — NeuronFS 엔진을 만들 도구
    터미널에 go version 쳐서 1.22 이상이면 OK
💡
Go가 없다면? go.dev/dl에서 다운로드하세요. 설치 후 터미널을 다시 열면 됩니다.
1
다운로드 & 빌드
코드를 받고 실행 파일을 만듭니다 (30초)

터미널(명령 프롬프트)을 열고 아래 명령어를 한 줄씩 입력하세요.

# 1. NeuronFS 코드 다운로드
$ git clone https://github.com/rhino-acoustic/NeuronFS.git

# 2. 엔진 폴더로 이동
$ cd NeuronFS/runtime

# 3. 실행 파일 빌드 (4MB짜리 파일이 생깁니다)
$ go build -o neuronfs .
# 1. NeuronFS 코드 다운로드
PS> git clone https://github.com/rhino-acoustic/NeuronFS.git

# 2. 엔진 폴더로 이동
PS> cd NeuronFS\runtime

# 3. 실행 파일 빌드
PS> go build -o neuronfs.exe .
ℹ️
끝나면 neuronfs (또는 neuronfs.exe) 파일이 생깁니다. 이것이 NeuronFS 엔진입니다. 딱 4MB. 다른 건 아무것도 필요 없습니다.
2
나만의 뇌 만들기
빈 뇌를 초기화합니다 (5초)

사람의 뇌에 7개 영역이 있듯이, NeuronFS의 뇌에도 7개 영역이 있습니다. 이 명령어 하나로 전부 자동 생성됩니다.

$ ./neuronfs --init ./my_brain

🧬 NeuronFS initialized: ./my_brain
├── brainstem/ ← P0: 절대 원칙 (가장 강력)
├── limbic/ ← P1: 감정
├── hippocampus/ ← P2: 기억
├── sensors/ ← P3: 환경
├── cortex/ ← P4: 지식 (여기에 코딩 규칙!)
├── ego/ ← P5: 성격
└── prefrontal/ ← P6: 목표
🎮
디아블로 2를 해봤다면: P0 = Zod 룬, P6 = El 룬이라고 생각하세요. 낮은 숫자가 항상 이깁니다.
3
첫 번째 규칙 만들기
AI에게 "console.log 쓰지 마"라고 각인시킵니다

禁은 "절대 하지 마"라는 뜻의 한자입니다. 이 글자로 시작하는 폴더를 만들면 AI가 절대로 그 행동을 하지 않습니다.

# "console.log 절대 금지" 규칙 생성
$ ./neuronfs ./my_brain --grow cortex/coding/禁console_log

✅ neuron created: cortex/coding/禁console_log

하나 더 만들어볼까요? 必은 "반드시 해"라는 뜻입니다:

# "에러 처리 반드시 하기" 규칙 생성
$ ./neuronfs ./my_brain --grow cortex/coding/必error_handling

✅ neuron created: cortex/coding/必error_handling
my_brain/cortex/coding/
├── 禁console_log/ NEW
│ └── 1.neuron
└── 必error_handling/ NEW
    └── 1.neuron
⚡
이게 끝입니다. 폴더 하나가 곧 규칙 하나. 텍스트 파일에 규칙을 쓰지 않아도 됩니다. 폴더 이름이 곧 규칙입니다.
4
모든 AI에 배포하기
뇌를 시스템 프롬프트로 변환합니다 (1초)

이 명령어 하나로 4개 AI 도구에 동시 배포됩니다:

$ ./neuronfs ./my_brain --emit all

[EMIT] ✅ Cursor → .cursorrules
[EMIT] ✅ Claude → CLAUDE.md
[EMIT] ✅ Gemini → ~/.gemini/GEMINI.md
[EMIT] ✅ Copilot → .github/copilot-instructions.md

✅ 4 targets written. 2 neurons active.

🎉 끝났습니다! 이제 어떤 AI를 쓰든 같은 규칙이 적용됩니다.

5
더 해보기
여기부터는 선택사항입니다

기본은 끝났습니다. 더 재밌는 것들:

# 규칙의 중요도 높이기 (+1 가중치)
$ ./neuronfs ./my_brain --fire cortex/coding/禁console_log

# 3D 대시보드로 뇌 시각화
$ ./neuronfs ./my_brain --api
→ localhost:9090 에서 확인

# AI가 자동으로 규칙을 진화시키기
$ ./neuronfs ./my_brain --evolve
→ Groq LLM이 교정 로그를 분석하여 새 규칙 제안
🔮
vorq가 뭔가요? 이 단어는 어떤 사전에도 없습니다. AI가 이 단어를 보면 "이건 뭐지?" 하고 반드시 찾아봅니다. 이것이 NeuronFS가 AI를 100% 통제하는 비밀입니다. 자세히 보기 →

추천: Antigravity (Gemini) + NeuronFS
왜 이 조합이 바이브코더에게 최강인가

Antigravity는 Google의 AI 코딩 어시스턴트입니다. NeuronFS와 함께 쓰면:

🧠
NeuronFS MCP 서버
Antigravity가 자동 연결.
grow, fire, signal을 IDE 안에서 바로 실행
🌍
Google 생태계
Gemini 2.5 Pro · 100만 토큰 · 무료 티어
🎨
초보자 친화 UI
VS Code 안에서 대화하듯 코딩.
터미널 공포 없음
⚡
바이브코딩 최적
"이렇게 해줘" → AI가 코딩.
NeuronFS가 품질 보장
🐣
바이브코더란? 코드를 직접 쓰지 않고 AI에게 "이렇게 만들어줘"라고 말하는 사람.
문제: AI가 규칙을 자꾸 어긴다. 해결: NeuronFS가 규칙을 구조로 강제한다.
Antigravity + NeuronFS = AI가 코딩하고, 폴더가 품질을 보장한다.
# Antigravity에서 NeuronFS MCP 서버 연결 후
