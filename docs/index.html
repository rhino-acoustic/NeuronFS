Thesis

The right question matters more than the right answer

1. The first executable format for values

Humanity has stored values for millennia — in law, scripture, policy — but could never execute them directly. Brain folders are the first format where values are read by humans and executed by machines without interpretation.

2. Model-independent knowledge asset

GPT → Claude → Gemini → whatever comes next. The brain stays. Organizational judgment accumulates independently of any AI vendor. Knowledge decoupled from the model for the first time.

3. Accountability infrastructure for the AI era

"Why did the AI make that decision?" The era of proving AI decisions is coming. Brain folders + git logs are the only auditable, pre-documented governance structure. Not XAI (explaining the model) — documenting the rules given to it.

4. Preventing the death of expert knowledge

When a 30-year veteran retires, their judgment criteria vanish. Brain folders preserve decision-making structures in an inheritable, executable format. The brain as a testament.

5. The standard that doesn't exist yet

CLAUDE.md, AGENTS.md, .cursorrules — the industry is converging on file-based AI governance. But there's no standard format for AI operational knowledge. Whoever defines that format shapes the next era.

Features

Why folders beat vectors

NeuronFS is a prompt governance tool that structures and prioritizes AI rules via the filesystem. The prompt generation is deterministic; the LLM's response to that prompt is probabilistic.

📁

Folders = Rules

mkdir a rule. rm a rule. git diff your governance. Every change is auditable, version-controlled, and human-readable. Learn more →

⚡

Fast Lookup

Individual rules resolve via the OS dentry cache. The full emit pipeline walks all neurons (O(n)), but with 420 neurons completes in <40ms. Learn more →

🔒

Air-Gap Ready

No network calls. No telemetry. Runs from a USB stick. Suitable for restricted environments where data cannot leave the premises. Learn more →

🎯

Multi-Emit

One brain, six outputs: GEMINI.md, .cursorrules, CLAUDE.md, AGENTS.md, COPILOT.md, .windsurfrules. Same governance, every tool.

🧬

Self-Evolution

Hebbian learning: neurons that fire get reinforced. TTL decay: unused neurons fade. Closed Learning Loop: skills auto-accumulate.

💰

$0 Forever

No cloud. No subscription. No vendor lock-in. A single 4MB Go binary. AGPL-3.0 open source. Minimal infrastructure compared to vector DB alternatives.

Architecture

7-Layer Subsumption Cascade

Inspired by Rodney Brooks' robotics architecture (1986). Lower layers take structural precedence in the emit pipeline — conceptually analogous to hardware interrupts.

When brainstem (P0) contains a 禁 folder, the emit pipeline places it as a NEVER-tier rule. Higher layers cannot weaken it at the prompt generation level.

Honest Note
This is a prompt generation tool. The subsumption cascade controls what appears in the output — but final LLM behavior remains probabilistic.
P0Brainstem禁/bomb — NEVER-tier prohibitions
P1LimbicEmotionPrompt — focus, stress signals
P2HippocampusMemory, skills, episodic recall
P3SensorsExternal input, NAS watchers
P4CortexCode rules, codemap, error patterns
P5EgoPersona, language, identity
P6PrefrontalStrategic direction, long-term goals

Runewords

Governance Opcodes

Han ideograph characters serve as single-character opcodes. Each maps to a specific enforcement behavior in the 3-tier system: ALWAYS, WHEN→THEN, NEVER.

The analogy is from Diablo II runewords: the folder is the socket, the opcode is the rune. The combination creates deterministic governance at the prompt generation level.

Limitations

What NeuronFS is and is not

NeuronFS ISNeuronFS IS NOT
A prompt structuring toolA model modification system
A filesystem-based rule organizerA vector DB replacement for all uses
A governance framework for AI agentsAn inference-level enforcement layer
A way to improve instruction adherenceA guarantee of 100% compliance

NeuronFS controls the input to AI tools with structural precision. The output remains probabilistic.

A 禁 folder guarantees the NEVER-tier rule appears in the prompt. It does not guarantee the model will never violate it. In practice, well-structured prompts significantly reduce violations.
