Abstract

We present NeuronFS, a filesystem-native cognitive architecture that achieves deterministic AI agent governance with zero infrastructure dependency. Unlike probabilistic approaches (vector databases, embedding-based retrieval), NeuronFS encodes governance rules as OS directory structures — achieving O(1) rule lookup via the kernel's dentry cache. A 4MB Go binary walks the folder hierarchy, resolves a 7-layer subsumption cascade inspired by Brooks' robotics architecture, and emits format-specific instruction files for any AI tool (Cursor, Claude, Gemini, GPT). In controlled benchmarks, NeuronFS-harnessed agents show a +43% performance improvement over bare models, with an 8.9GB model (Gemma 4) outperforming a bare 70GB model (Llama 3). The system has been validated over 13 evolutionary versions with 420+ active neurons and 9,100+ total activations.

1. Introduction

The dominant paradigm for AI agent memory and governance relies on embedding vectors stored in external databases (Pinecone, Chroma, Weaviate). This approach suffers from three fundamental limitations: non-determinism (similarity search returns probabilistic results), infrastructure dependency (requires cloud services), and opacity (vectors are not human-readable).

NeuronFS inverts this paradigm entirely. Instead of storing rules as embeddings in a database, rules are expressed as folders in the OS filesystem. The folder name is the rule. The folder depth is the specificity. The folder's parent path is the context. This approach leverages the most ancient, battle-tested data structure in computing — the B-tree directory index — for a problem that the industry has been overengineering with billion-dollar infrastructure.

Key Insight
mkdir brain/brainstem/禁/data_leak — This single command creates a prohibition rule that is structurally encoded in the filesystem and emitted as a highest-priority NEVER-tier instruction. Unlike ad-hoc prompt text, this rule is structurally guaranteed to appear in the output — though, like all prompt-based governance, final adherence depends on the LLM's probabilistic interpretation.

The philosophical foundation is simple: "When a document becomes a folder, hierarchy is born." A non-developer listed philosophical declarations as folder names. Those declarations became circuits. The result: a 4MB binary that governs AI agents with zero tokens consumed, zero API calls, and zero infrastructure.

2. Problem Statement

Current AI governance approaches exhibit the following failure modes:

2.1 The Probabilistic Trap

System prompts and RAG-based rules are processed by the language model itself. The model interprets the rules — meaning it can ignore them, misinterpret them, or hallucinate alternative rules. This is inherently probabilistic. A rule that works 95% of the time is not governance — it is a suggestion.

2.2 Infrastructure Overhead

A typical vector-based governance stack requires: a vector database ($70–200/month), an embedding model API ($0.0001/token), a retrieval pipeline, and a production server. NeuronFS requires: a USB stick. The total cost of the most sophisticated AI governance system in the world should not exceed the price of a USB drive.

2.3 The Context Window Illusion

Even with 1M+ token context windows, system prompts suffer from positional decay — instructions placed early in the prompt are progressively ignored. NeuronFS sidesteps this entirely by encoding rules structurally, outside the token stream, where they cannot be subject to attention degradation.

Industry Observation
The industry spent billions making models smarter. NeuronFS asks: what if we made the rails perfect instead? A dumb engine on perfect rails always arrives. A smart engine on broken rails derails.

3. Subsumption Cascade Architecture

The brain is organized into 7 layers, inspired by Rodney Brooks' subsumption architecture in robotics (1986). Each layer has a priority level (P0–P6). Lower layers take structural precedence over higher layers in the emit pipeline — conceptually analogous to a hardware interrupt model, though the enforcement occurs at the prompt generation stage, not at the model inference stage.

P0Brainstem禁/bomb — Highest-priority prohibitions. Emitted as NEVER-tier rules. Structurally placed first in all outputs.
P1LimbicEmotionPrompt engine. Focus signals, stress adaptation, dopamine-driven reinforcement.
P2HippocampusPersistent memory. Learned skills (Closed Learning Loop). Episodic recall. Spaced repetition.
P3SensorsExternal input: NAS filesystem watchers, brand guidelines, market data connectors.
P4CortexDevelopment rules, codemap, error pattern database, architecture constraints.
P5EgoPersona definition. Language preference. Communication tone. Identity expression.
P6PrefrontalStrategic direction. Long-term objectives. Project-level planning.
Figure 1. The 7-layer subsumption cascade. P0 (brainstem) always overrides P6 (prefrontal). A bomb neuron at P0 halts the entire system — no higher layer can resume operation.

3.1 The Override Mechanism

When the emit pipeline walks the brain folder, it processes regions in P0→P6 order. If a 禁 (prohibition) folder is found in brainstem (P0), the rule is emitted as a NEVER tier — placed in the highest-priority section of the output. Even if cortex (P4) contains a contradictory 推 (recommendation), the brainstem rule takes absolute precedence. This is structurally guaranteed by the traversal order, not by model interpretation.

3.2 Emission Tiers

The emit pipeline produces three tiers of rules, each mapped to the subsumption cascade:

  • ALWAYS: Rules that fire on every invocation. Brainstem prohibitions, mandatory checks.
  • WHEN → THEN: Conditional triggers. "When modifying Go code, fire the codemap neuron."
  • NEVER: Absolute prohibitions. Derived from 禁 folders. Cannot be weakened by any mechanism.

The emit pipeline is the core transformation engine. It converts a filesystem hierarchy into format-specific instruction files consumed by AI tools.

6. Self-Evolution Mechanisms

6.1 Hebbian Learning

"Neurons that fire together, wire together." Each neuron has a counter field that increments when the neuron is activated (read, referenced, or triggered). Neurons with high activation counts are promoted to "강화" (reinforced) status, receiving priority in emit output. Neurons that remain dormant undergo TTL decay and are eventually archived.

6.2 Closed Learning Loop (Phase 58)

Inspired by Hermes Agent's skill accumulation pattern, NeuronFS now implements a closed learning loop: successful operations generate .skill files in hippocampus/skills/. These skills are automatically injected into the cortex emit pipeline, enabling the agent to leverage past success patterns in future sessions.

flowClosed Learning Loop
Build Success → LearnSkill() → .skill file → hippocampus/skills/ ↓ Next Session ← InjectSkillsToPrompt() ← emit pipeline ← _rules.md

6.3 Dopamine & Bomb Signals

Positive outcomes trigger dopamine signals, reinforcing the activated neuron path. Catastrophic failures trigger bomb signals at P0, creating permanent prohibition neurons that prevent the same failure pattern from recurring.

9. Use Cases

9.1 Enterprise AI Governance

Deploy standardized AI rules across 1,000+ developers. A single brain folder, distributed via git or USB, ensures every AI agent in the organization adheres to the same governance framework. No cloud dependency. Compliance-ready. SOC 2 and HIPAA compatible by design — no data leaves the premises.

9.2 Air-Gapped Environments

Military, healthcare, and financial institutions operate in environments where data cannot leave the premises. NeuronFS runs entirely offline. Zero network calls. Zero telemetry. Zero data exfiltration risk. The entire brain can be encrypted with XChaCha20-Poly1305 and transported on a single USB device.

9.3 Developer Productivity

Teach your AI coding assistant project-specific rules: naming conventions, architecture patterns, forbidden anti-patterns. Skills accumulate over time through the Closed Learning Loop, meaning the governance system becomes more effective the longer it is used.

9.4 AI Education & Research

NeuronFS provides a tangible, hands-on system for teaching cognitive architecture concepts. Students mkdir neurons and observe subsumption in real-time. The filesystem is the curriculum. No frameworks to learn. No APIs to configure. Just folders and files.

11. Limitations & Scope

Important Disclaimer
NeuronFS is a prompt generation tool, not an AI model modification system. It structures and prioritizes governance rules in the emitted output, but final enforcement depends on the LLM's interpretation of those rules. No prompt-based system can guarantee 100% instruction adherence due to the probabilistic nature of language models.

11.1 Enforcement Boundary

NeuronFS controls the input to AI tools with high precision — ensuring rules appear, are correctly prioritized, and are structurally organized. However, the output of the AI model remains probabilistic. A 禁 folder guarantees the NEVER-tier rule will be present in the prompt; it does not guarantee the model will never violate that rule. In practice, well-structured prompts significantly reduce violation rates, but the theoretical guarantee is at the prompt level, not the inference level.

11.2 Benchmark Methodology

The benchmark results (Section 7) were conducted using a 40-item instruction adherence evaluation suite designed to test rule following across code generation, format compliance, and constraint satisfaction tasks. The tests were run by the project maintainer using consistent settings. These results have not been independently reproduced. We encourage the community to replicate and critique these benchmarks. The test methodology and prompts will be published on the project wiki.

11.3 What NeuronFS Is and Is Not

NeuronFS ISNeuronFS IS NOT
A prompt structuring and management toolA model modification or fine-tuning system
A filesystem-based rule organization systemA vector database replacement for all use cases
A governance framework for AI coding assistantsA security enforcement layer at the inference level
A way to improve instruction adherence through better promptsA guarantee of 100% rule compliance

The architectural analogies used throughout this document (subsumption cascade, hardware interrupts, runewords) are conceptual models that describe the structure and priority system of the emit pipeline. They help explain the design philosophy but should not be interpreted as literal claims about OS-level enforcement of AI behavior.
