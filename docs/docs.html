Subsumption Cascade Architecture

The brain is organized into 7 layers, inspired by Rodney Brooks' subsumption architecture in robotics (1986). Each layer has a priority level (P0–P6). Lower layers take structural precedence over higher layers in the emit pipeline — conceptually analogous to a hardware interrupt model, though the enforcement occurs at the prompt generation stage, not at the model inference stage.

The 7 Layers

P0Brainstem禁/bomb — NEVER-tier prohibitions. Structurally first.
P1LimbicEmotionPrompt engine. Focus, stress adaptation.
P2HippocampusMemory, skills, episodic recall, spaced repetition.
P3SensorsExternal input: NAS watchers, brand guides, data.
P4CortexCode rules, codemap, error patterns, skills.
P5EgoPersona, language preference, identity.
P6PrefrontalStrategic direction, long-term goals.

How the emit pipeline works

The Go binary (neuronfs) walks the brain_v4/ directory tree. For each folder:

  1. Detect opcode (禁, 必, 推) from folder name or content
  2. Assign to 3-tier system: ALWAYS, WHEN→THEN, NEVER
  3. Resolve subsumption: P0 rules override P6 rules in output position
  4. Emit format-specific files: GEMINI.md, .cursorrules, CLAUDE.md, etc.

The full pipeline walks all neurons (O(n) where n = neuron count). With 420 neurons, this completes in ~40ms on commodity hardware.

Governance Opcodes (Runewords)

Han ideograph characters serve as single-character opcodes that map to specific enforcement behaviors. The analogy draws from Diablo II runewords — the folder is the socket, the opcode is the rune.

Note: "Runeword" and "dopamine" are conceptual metaphors for the priority and reinforcement systems, not literal claims about system behavior.

3-Tier enforcement system

The emit pipeline organizes all rules into three tiers:

  • ALWAYS — Applied every response, unconditionally (必 opcodes)
  • WHEN→THEN — Conditional triggers that activate based on context (推 opcodes)
  • NEVER — Absolute prohibitions, placed first in output (禁 opcodes)

This 3-tier system ensures the emitted prompt is well-structured, but final LLM adherence is probabilistic.

Self-Evolution Mechanisms

Hebbian Reinforcement

Neurons that are activated (referenced in corrections or successful outcomes) get their activation count incremented. Higher activation = higher precedence within the same priority tier.

TTL Decay

Neurons with a TTL (Time-To-Live) frontmatter field decay over time. Unused neurons naturally fade, keeping the brain lean. Neurons below threshold enter "probation" before removal.

Closed Learning Loop

Closed Learning Loop flow
Build Success → LearnSkill() → .skill file → hippocampus/skills/ ↓ Next Session ← InjectSkillsToPrompt() ← emit pipeline ← _rules.md

Successful operations generate .skill files in hippocampus/skills/. These are automatically injected into future emit cycles, creating a feedback loop. Inspired by the Hermes Agent skill accumulation pattern.

Corrections & Bomb signals

Negative outcomes are recorded in corrections.jsonl. After 3 consecutive failures on the same pattern, a bomb signal is created at P0 (brainstem), creating a permanent prohibition neuron.

Positive outcomes trigger dopamine signals (conceptual metaphor for positive reinforcement), reinforcing the activated neuron path.

Limitations & Scope

Important Disclaimer
NeuronFS is a structured prompt generation tool, not an AI model modification system. The title "AI Governance" refers to governance of the prompt — not control of the model's inference. Final LLM adherence to any prompt-based rule is probabilistic.

What NeuronFS Is and Is Not

NeuronFS ISNeuronFS IS NOT
A structured prompt generation toolA model modification or fine-tuning system
A filesystem-based rule organizerA vector DB replacement for all use cases
A governance framework for prompt structureAn inference-level enforcement layer
A way to improve instruction adherenceA guarantee of 100% rule compliance

Known Limitations

  • "Deterministic" applies to the prompt generation, not the LLM response. A more precise term would be "Structured Prompt Generation."
  • O(1) vs O(n): Individual dentry lookups are O(1), but the full emit pipeline walks all n neurons — making the overall process O(n).
  • Infrastructure: A 4MB binary + filesystem is minimal infrastructure, not truly "zero." This is relative to cloud-based alternatives.
  • Benchmarks: Self-designed, self-administered. Independent peer review has not been conducted.
  • Terminology: "Runeword," "dopamine signal," "subsumption cascade" are conceptual metaphors that describe the system's design philosophy.
